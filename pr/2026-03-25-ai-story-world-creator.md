# AI 故事世界共创师

> **一句话卖点**：下班后躺在沙发上的职场人，用 AI 10 秒进入角色扮演世界，从被动刷 2 小时视频看完就忘，到主动创造属于自己的故事世界

## 概述

基于 Issue #149 创建的详细设计文档。

现代人的娱乐方式陷入「被动消费」陷阱：刷 2 小时短视频，笑完就忘；想玩桌游 RPG 找不到队友；想体验剧情游戏受限于固定脚本。结果是：娱乐时间过去了，但什么都没留下，反而更空虚。

本产品通过自然语言角色扮演 + 动态世界生成 + 多角色互动，让用户从「消费故事」变成「创造故事」，每次冒险都是独一无二的体验。

## 功能设计

### 核心功能

1. **自然语言角色扮演**
   - 直接对话，无需学习规则
   - AI 理解用户意图并推进剧情
   - 支持任意角色设定（末世机械师、武侠剑客、星际船长...）

2. **动态世界生成**
   - 用户说「我想当海盗」→ 10 秒进入海盗船场景
   - AI 根据用户选择实时调整世界观和剧情走向
   - 每次选择都有蝴蝶效应，故事永不重复

3. **多角色互动**
   - AI 扮演多个 NPC，每个都有独特性格和背景
   - NPC 之间有互动，世界有生命力
   - 支持「剧情导演」模式，协调 NPC 行为避免矛盾

4. **故事保存分享**
   - 完成的冒险保存为「故事书」
   - 支持导出 Markdown/JSON 格式
   - 朋友可以续写你的故事

5. **多种题材**
   - 内置题材：末世生存、武侠江湖、科幻星际、悬疑推理、恋爱养成
   - 支持自定义世界观

6. **「随时暂停」机制**
   - 现实中有事可以随时暂停
   - 回来时「前情提要」快速回忆

### 差异化功能

| 现有方案 | 本产品 |
|---------|--------|
| 刷视频 2 小时，看完就忘 | 创作自己的故事，每次都不同 |
| 想玩桌游找不到人 | AI 24 小时在线，随时开始 |
| 剧情游戏固定分支 | AI 实时生成，无限可能 |
| 只能消费不能创造 | 边玩边创，故事属于你 |

## 技术方案

### 数据层

```
故事数据：
- 世界观设定（主题、规则、NPC）
- 故事状态（当前进度、角色关系、历史事件）
- 用户选择历史（决策点 + 后果）

用户数据：
- 偏好设置（喜欢的题材、难度等级）
- 历史故事库（已完成的冒险）
- 社交数据（分享的故事、续写的章节）

内容数据：
- 预设题材模板（末世、武侠、科幻...）
- NPC 角色库（性格向量、对话风格）
- 场景库（描述模板、事件触发器）
```

### 逻辑层

```
1. 故事引擎模块
   - 世界观状态管理（维护一致性）
   - 剧情推进逻辑（基于用户选择）
   - 一致性检查器（检测逻辑漏洞）

2. 角色扮演模块
   - 多 Agent 架构（每个 NPC 一个 Agent）
   - 性格向量系统（决定 NPC 行为）
   - 剧情导演（协调 NPC 互动）

3. 动态生成模块
   - 场景生成（LLM + 模板）
   - 事件生成（随机 + 剧情驱动）
   - 对话生成（基于角色 + 情境）

4. 难度自适应模块
   - 新手：AI 主动引导，选项明确
   - 老手：开放世界，自由探索
   - 根据用户历史行为自动调整
```

### 展示层

```
移动端（主场景）：
- 首页：快速开始（「今晚想成为谁？」）
- 对话界面：沉浸式角色扮演
- 故事书：已完成的冒险
- 分享：故事导出 + 续写

Web 端（管理）：
- 题材商店：浏览/购买新题材
- 社区广场：发现其他创作者的故事
- 创作者工具：自定义世界观
```

## 实现步骤

### Phase 1：MVP 验证（2 周）

1. **核心对话系统**
   - 单一题材（推荐：末世生存或武侠江湖）
   - 基础角色扮演对话
   - 简单分支选择

2. **基础世界生成**
   - 3-5 个预设场景
   - 2-3 个 NPC
   - 简单剧情线

3. **用户验证**
   - 邀请 100 名种子用户
   - 收集反馈，验证 PMF

### Phase 2：多题材扩展（2 周）

1. **题材系统**
   - 支持多题材切换
   - 世界观自定义
   - NPC 角色库扩展

2. **故事保存**
   - 故事书导出
   - Markdown/JSON 格式

### Phase 3：多角色互动（2 周）

1. **多 Agent 架构**
   - 多个 NPC 同时在线
   - NPC 之间互动
   - 情感记忆系统

2. **难度自适应**
   - 新手/老手模式
   - 自动难度调整

### Phase 4：社交功能（2 周）

1. **故事分享**
   - 社区广场
   - 故事续写
   - UGC 分成机制

2. **付费功能**
   - 高级题材
   - 语音演绎
   - 无限创作

## 资源需求

### API

- **LLM API**：GPT-4 / Claude / kimi-k2.5
  - 用途：对话生成、场景生成、剧情推进
  - 预估成本：¥0.05-0.15/对话轮次
  - 优化方案：本地小模型 + 云端大模型混合

- **语音 API**（可选）：ElevenLabs / Azure TTS
  - 用途：AI 语音演绎
  - 预估成本：¥0.02/秒

### 服务

- **后端**：Node.js / Python
- **数据库**：PostgreSQL（故事状态） + Redis（会话缓存）
- **存储**：S3（故事书导出）
- **实时通信**：WebSocket（对话流）

### 开发团队

- **MVP 阶段**：2 人（1 后端 + 1 前端）
- **扩展阶段**：3-4 人（+1 Prompt 工程师）

## 变现模式

### Freemium 模式

- **免费版**：
  - 每月 3 个免费故事
  - 基础题材（末世、武侠）
  - 文字对话

- **订阅版（¥19.9/月）**：
  - 无限创作
  - 所有题材解锁
  - 语音演绎
  - 优先响应速度

### 单次付费

- **故事包**：¥9.9/主题包（科幻、悬疑、恋爱...）
- **语音增值**：¥4.9/月（AI 语音演绎）

### UGC 分成

- 用户创作的故事上架
- 平台抽成 30%
- 创作者获得 70% 收益

### IP 孵化（长期）

- 优秀故事改编为小说/游戏/影视
- 与创作者分成

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| **内容合规**：用户可能生成敏感/违规内容 | AI 内容过滤 + 用户举报机制 + 人工审核 |
| **故事崩坏**：LLM 忘记设定，剧情矛盾 | 故事一致性检查器 + 状态管理 + 关键事件记忆 |
| **用户留存**：故事完结后流失 | 「无尽模式」+ 定期更新官方故事线 + 社区续写 |
| **计算成本**：长篇对话消耗大量 token | 本地小模型 + 云端大模型混合 + 对话压缩 |
| **竞品分流**：Character.AI、Dreamily 已有类似功能 | 聚焦「完整故事体验」而非「角色聊天」+ 中文题材差异化 |
| **内容同质化**：AI 生成故事模式化 | Prompt 工程优化 + 多样性注入 + 用户反馈循环 |

---

**为什么现在做？** GPT-4/Claude 的角色扮演能力已经非常强，多模态（语音+图像）让沉浸感大幅提升。这是技术成熟的最佳时机。

**为什么聚焦单一题材？** 一个题材的成功比十个题材的平庸更有价值。推荐先做「末世生存」或「武侠江湖」——中国用户接受度高，验证 PMF 后再扩展。
