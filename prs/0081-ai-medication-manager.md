# AI 智能用药管家

> **一句话卖点**：独居老人对着药瓶拍张照，AI 自动生成用药时间表，每天语音提醒服药，子女手机同步收到确认通知——多重用药不再混乱危险

## 概述

基于 Issue #81 创建的详细设计文档。

针对 65 岁以上老人平均服用 5+ 种药物的现状，打造「拍照识别 + 智能提醒 + 家人同步」三重守护系统。解决老人记不住、看不清、吃不对的问题，让子女远程也能安心。

## 功能设计

### 核心功能

1. **药品智能识别**
   - 拍照识别药瓶包装（OCR + 图像识别）
   - 自动提取药物名称、用法用量、注意事项
   - 支持中文药瓶、英文进口药

2. **智能用药提醒**
   - 语音播报「该吃药了」+ 手机震动
   - 大字界面显示今日待服药物
   - 扫码/拍照确认已服用

3. **用药日历可视化**
   - 每日/每周用药计划一目了然
   - 历史服药记录追溯
   - 支持导出给医生查看

4. **家人远程同步**
   - 子女手机实时查看父母用药情况
   - 漏服/误服自动推送告警
   - 支持多个家庭成员绑定

5. **药物冲突预警**
   - 检测药物相互作用（如头孢 + 酒精）
   - 提示重复成分（多种感冒药混吃风险）
   - 显示冲突严重程度和替代建议

### 辅助功能

- 语音助手：「今天还有哪些药没吃？」
- 药品库存提醒：「降压药还剩 3 天，需要补货」
- 处方药有效期追踪

## 技术方案

### 数据层

```
药物数据库
├── 基础信息（名称、厂商、规格）
├── 用法用量（成人/儿童剂量）
├── 相互作用知识库
└── 用户个性化用药记录
```

**数据来源**：
- 药监局公开数据爬取
- DrugBank 开源相互作用库
- 用户众包校验（标注换会员）

### 逻辑层

```
[拍照输入] → OCR 识别 → 药物匹配 → 用法提取
                              ↓
[用户画像] ← 个性化配置 ← 冲突检测
                              ↓
[定时任务] → 语音合成 → 推送提醒 → 家人同步
```

**核心技术栈**：
- OCR：百度/腾讯云药瓶场景优化 API
- 冲突检测：DrugBank + 自建规则引擎
- 语音合成：TTS API（科大讯飞/阿里云）
- 推送：APNs + FCM

### 展示层

**老人端**（极简设计）：
- 超大字体（30px+）
- 高对比度配色
- 语音交互优先
- 单击确认服药

**子女端**（信息丰富）：
- 用药日历视图
- 异常告警通知
- 父母用药趋势分析
- 一键呼叫药店配送

## 实现步骤

### Phase 1: MVP（2 周）

1. 搭建基础框架（React Native）
2. 实现拍照识别（OCR API 集成）
3. 手动录入用药计划
4. 简单定时提醒（本地通知）

**验证目标**：老人能独立完成拍照 + 确认服药流程

### Phase 2: 核心功能（4 周）

1. 药物数据库构建（常见 500 种）
2. 自动提取用法用量
3. 语音提醒 + 大字界面
4. 家人同步功能

**验证目标**：老人连续 7 天正确服药率 > 90%

### Phase 3: 智能化（4 周）

1. 药物冲突检测
2. 库存智能提醒
3. 用药趋势分析
4. 处方药有效期追踪

**验证目标**：成功预警 3 起潜在用药风险

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| OCR 识别错误导致服错药 | 高 | 多重校验（名称 + 厂商 + 批准文号），低置信度时人工确认 |
| 药物数据库不全 | 中 | 先聚焦常见 500 种，众包补全长尾 |
| 老人不会用智能手机 | 高 | 极简 UI + 语音交互 + 子女代设置 |
| 医疗责任边界模糊 | 高 | 明确「提醒工具」定位，不替代医嘱，免责声明 + 保险 |
| 用户冷启动流失 | 中 | 7 天新手引导 + 客服电话支持 |
