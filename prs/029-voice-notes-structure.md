# AI 语音笔记结构化助手

> 让零散的语音想法自动井井有条

## 功能设计

### 核心功能

1. **智能分类引擎**
   - 自动识别内容类型：想法/待办/会议记录/灵感/备忘
   - 多标签支持：一条笔记可同时属于多个类别
   - 置信度评分：当分类不确定时标注，供用户确认

2. **结构化输出**
   - 自动生成标题：从散乱语音中提取核心主题
   - 智能标签：基于内容关键词和上下文
   - 优先级推断：从语气词和时间敏感词判断紧急程度
   - 格式化输出：Markdown / Notion / Obsidian 格式

3. **知识关联网络**
   - 检测与已有笔记的语义关联
   - 自动建立双向链接
   - 标记重复或相关主题
   - 构建个人知识图谱

4. **情感线索保留**
   - 识别语音中的情绪标记（兴奋/犹豫/紧急/困惑）
   - 生成情感标签用于回顾
   - 情绪趋势分析

5. **智能提醒系统**
   - 自动识别待办事项并推送
   - 集成日历 / 任务管理工具
   - 基于紧急程度和截止日期的智能提醒

## 技术方案

### 数据层

```yaml
语音输入:
  - 格式: WebM / M4A / WAV
  - 存储: 云端临时存储，处理后删除

笔记数据:
  - 主库: PostgreSQL / Supabase
  - 向量库: Pinecone / Qdrant (语义搜索)
  - 缓存: Redis (热点数据)

用户配置:
  - 偏好设置: 笔记格式、默认分类
  - 集成配置: Notion/Obsidian/日历 API keys
```

### 逻辑层

```yaml
语音处理:
  ASR: 
    - 首选: Whisper Large V3 (高精度)
    - 备选: 讯飞语音 (中文优化)
  
  情感分析:
    - Whisper 的时间戳 + 音高/语速特征
    - 情感分类模型 (兴奋/犹豫/紧急/平静)

文本处理:
  分类器:
    - LLM: GPT-4 / Claude 3.5
    - Prompt: 结构化分类 + 置信度输出
  
  结构化:
    - 标题生成: LLM 提取核心主题
    - 标签提取: NER + 关键词 + LLM
    - 优先级: 规则 + LLM 判断
  
  关联检测:
    - 向量嵌入: text-embedding-3-small
    - 相似度阈值: 0.75
    - 图数据库: Neo4j (可选，用于复杂关联)

集成层:
  Notion API: 笔记同步
  Obsidian: Local REST API 插件
  日历: Google Calendar / Apple Calendar
```

### 展示层

```yaml
Web 端:
  - React / Next.js
  - 实时转写显示
  - 笔记编辑器 (支持 Markdown)

移动端:
  - PWA / React Native
  - 语音录制组件
  - 离线队列 (无网络时暂存)

桌面端:
  - Electron (可选)
  - 系统托盘快速录音
```

## 实现步骤

### Phase 1: MVP (2-3 周)

1. **基础语音转写**
   - 集成 Whisper API
   - 简单的 Web 录音界面
   - 转写结果展示

2. **核心分类**
   - 5 类基础分类 (想法/待办/会议/灵感/备忘)
   - LLM 分类 Prompt 设计
   - 基础结构化输出

3. **简单存储**
   - 笔记 CRUD
   - 用户认证
   - 基础搜索

### Phase 2: 智能化 (3-4 周)

4. **知识关联**
   - 向量嵌入集成
   - 语义相似度计算
   - 相关笔记推荐

5. **情感分析**
   - 语音情感特征提取
   - 情感标签生成
   - 情绪趋势可视化

6. **智能提醒**
   - 待办识别优化
   - 日历集成
   - 提醒规则引擎

### Phase 3: 集成 (2-3 周)

7. **外部工具集成**
   - Notion API 同步
   - Obsidian 插件

8. **移动端优化**
   - PWA 离线支持
   - 后台录音
   - 推送通知

### Phase 4: 增强 (持续)

9. **高级功能**
   - 风格学习 (学习用户表达习惯)

   - 团队协作
