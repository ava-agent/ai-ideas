# AI 书籍消化助手

> **一句话卖点**：书籍囤积族在书架前犹豫10分钟不知读哪本时，用 AI 书籍消化助手3秒获得个性化推荐，从买书如山倒到每周消化一本书

## 概述

基于 Issue #198 创建的详细设计文档。

**目标用户**：爱买书但读不完的知识工作者
**核心痛点**：书越囤越多，不知道从哪本开始，读了也记不住
**解决方案**：AI 驱动的书籍消化系统，从选择困难到知识内化

## 功能设计

### 核心功能

1. **书籍健康诊断**
   - 扫描书架（拍照/ISBN导入）
   - 基于用户兴趣和目标分析优先级
   - 生成「本周必读 Top 3」推荐

2. **15分钟精华提取**
   - 每章提取3个关键点
   - 支持多级摘要（3分钟/10分钟/完整版）
   - 根据用户目标个性化生成

3. **知识卡片生成**
   - 读后自动生成回顾卡片
   - 支持间隔重复算法推送
   - 可导出到笔记软件（Notion/Obsidian）

4. **跨书知识连接**
   - 自动识别书籍间的概念关联
   - 构建个人知识图谱
   - 推荐相关书籍和章节

5. **阅读动力追踪**
   - 进度可视化（已读60%）
   - 阅读打卡和成就系统
   - 社交分享（可选）

## 技术方案

### 数据层

**书籍数据库**
- 书籍元数据：ISBN、标题、作者、出版社、简介
- 章节结构：目录、章节标题、关键段落
- 知识标签：主题、概念、关键词

**用户数据**
- 书架清单：已购书籍列表
- 阅读进度：章节完成状态
- 知识卡片：生成的摘要和笔记
- 用户画像：兴趣、目标、阅读偏好

**存储方案**
- 书籍元数据：PostgreSQL
- 全文内容：向量数据库（Pinecone/Milvus）
- 用户数据：PostgreSQL + Redis（缓存）

### 逻辑层

**书籍解析服务**
```
输入：PDF/EPUB/MOBI 文件
处理：
  1. 格式转换 → 统一文本格式
  2. 结构识别 → 章节划分
  3. 关键段提取 → 重要内容标记
  4. 向量化 → 存入向量库
输出：结构化书籍数据
```

**摘要生成服务**
```
输入：章节内容 + 用户目标
处理：
  1. 关键点提取 → LLM 提取3个核心观点
  2. 个性化调整 → 根据用户画像优化
  3. 卡片生成 → 生成回顾卡片格式
输出：个性化摘要 + 知识卡片
```

**推荐算法**
```
输入：用户书架 + 阅读历史 + 当前目标
处理：
  1. 兴趣匹配 → 基于历史偏好
  2. 目标对齐 → 优先推荐与目标相关的书
  3. 难度评估 → 避免推荐太难或太简单
  4. 新鲜度 → 平衡新旧书籍
输出：Top 3 推荐列表 + 推荐理由
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 用户不打开 APP | 微信/邮件每日推送知识卡片，降低打开门槛 |
| 扫描版 PDF 识别率低 | 先支持电子书（epub/mobi），扫描版作为进阶功能 |
| 摘要质量参差不齐 | LLM + 人工校验混合模式，用户可反馈调整 |
| 版权问题 | 只处理用户合法购买的书籍，不提供全文分享 |
| 冷启动困难 | 预置热门书籍摘要，新用户可先体验再上传 |
| 付费意愿低 | 先做免费版积累用户，证明价值后再推付费 |
